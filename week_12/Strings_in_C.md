# String в С
За разлика от другите програмни езици, в С няма вграден тип за работа със символни низове или казано на *чист* български език, string-ове. В С те се представят като най-обикновени масиви съдържащи елементи от тип **char**. За разлика от масивите от тип някой от другите изучени вградени типове в С, когато инициализираме масив от тип **char** можем да напишем **char arr[] = "текст"**, където **текст** е съдържанието, което искаме нашият масив да съхранява.


<p>
  
```c
int main() {
  char str1[] = "Shrek"; // Инициализация
  return 0;
}
```
</p>

Също така, масивите от тип **char** могат да бъдат принтирани на конзолата без помощта на цикъл, използвайки функцията **printf()** и форматиращия модификатор **%s**.
<img width="1493" height="391" alt="image" src="https://github.com/user-attachments/assets/9fef280e-8c64-43a3-86b2-10f346773d8d" />

Сещате ли се как преди инициализирахме масиви? A как ги принтирахме?
<p>
  
```c
#include <stdio.h>

int main() {
  int arr[] = {1,2,3};
  return 0;

  for (int i = 0; i < 3; ++i)
    printf("%d ", arr[i]);
}
```
</p>

Можем да инициализираме и **char** масиви така, както го правим и за останалите типове, но тогава няма да можем да ги принтираме *готино* без цикъл.

<img width="1488" height="429" alt="image" src="https://github.com/user-attachments/assets/32245f81-0bf5-4ff8-ac01-e2819a6243ae" />

Забелязваме, че освен съдържанието на масива, на конзолата се принтират и някакви артефакти. Това се случва, защото програмата ни не знае кога да спре да чете записаното в паметта и почва да чете извън рамките на нашия масив. За да предотвратим това нашата програма да настъпва памет, която се намира извън рамките на нашия масив, трябва да поставим следния символ като последен елемент негов елемент **'\0'**. Този символ се нарича **терминираща нула** и тя обозначава края на един **string**. Когато инициализираме **string** по първия начин, който показахме, компилаторът поставя **терминиращата нула** вместо нас.
