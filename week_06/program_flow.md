## Ход на програма
Кодът, който до този момент писахме в рамките на курса изглеждаше по следния начин

```c
int main() {
  read_some_data; // line_1
  // proccess the data. Apply operations on it
  some_instruction;// line_2
  some_instruction;// line_3
  .
  .
  .some_instruction;// line_n
  return_some_data; // end
}
```
</p>

Такъв тип програми наричаме *процедурни*. Можем да си представим хода на тяхното изпълнение като една права. Всеки ред представлява една инструкция, която компютърът ни трябва да следва, за да изпълни дадена задача. Проблемът е, че за по- комплексни задачи кодът ни бързо нараства в обем, тъй като не разполагаме с инструменти за преизползването на някаква логика, която сме написали. Вече, може би, сте усетили колко е досадно да извършваме един и същи алгоритъм за няколко променливи от един и същи тип.

#### Пример:
Напишете програма, която въвежда 5 числа. Разделя всяко едно от тях на 5 с остатък и ги умножава по 3.

```c
int main() {
  int a, b, c, d, e;
  scanf("%d %d %d %d %d", &a, &b, &c, &d, &e);
  a = a % 5 * 3;
  b = b % 5 * 3;
  c = c % 5 * 3;
  d = d % 5 * 3;
  e = e % 5 * 3;
}
```
</p>

Ами, ако искаме програмата ни да прилага тази логика за произволен брой пъти **n** ? :D

## Конструкции за контрол на хода на програма в С (Control-flow structures)
През миналата седмица се запознахме с първите за нас такива конструкции, а именно **условните** (if, if-else, switch)
Те конструират "разклонения-пътеки" в хода на нашата програма.

## Тази седмица ще с още една фундаментална control-flow конструкция, а именно циклите
* for-loop
* while-loop
* do-while-loop
Те ще ни придоставят възможността да се връщаме назад в нашата програма и да изпълняваме даден алгоритъм, който сме написали, **последнователно И многократно** за различен вход от данни.
